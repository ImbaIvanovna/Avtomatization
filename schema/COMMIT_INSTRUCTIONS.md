## Как зафиксировать изменения в git

### Вариант 1: Использование готового скрипта

Запустите PowerShell скрипт из корня проекта:
```powershell
.\commit_changes.ps1
```

### Вариант 2: Ручной коммит

Если git установлен, выполните следующие команды:

```bash
# Инициализация репозитория (если еще не инициализирован)
git init

# Настройка пользователя
git config user.name "1Usinglable1"
git config user.email "ilya.razumov20022711@yandex.ru"

# Добавление всех изменений
git add .

# Создание коммита
git commit -m "Добавлено поле rating в таблицу records и выгружена актуальная схема БД

- Добавлено поле rating DECIMAL(3,2) в таблицу records
- Создана миграция migration_add_rating.sql
- Выгружена актуальная схема БД в schema/schema.sql
- Создан скрипт export_schema.py для выгрузки схемы"
```

### Вариант 3: Через Git Bash или другой интерфейс

Если git установлен, но не доступен через PowerShell, используйте Git Bash или другой интерфейс git и выполните команды из варианта 2.

## Проверка изменений

После коммита проверьте историю:
```bash
git log
```

Для просмотра изменений:
```bash
git status
git diff HEAD~1
```
